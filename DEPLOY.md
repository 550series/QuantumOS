# EdgeOne 部署指南

## 项目说明

这是一个基于 Next.js 14 的 MOSS 人工智能操作系统前端项目，使用 Tailwind CSS 进行样式设计。

## 构建项目

在部署到 EdgeOne 之前，需要先构建项目：

```bash
# 安装依赖
npm install

# 构建项目
npm run build
```

构建完成后，会在 `out` 目录生成静态文件。

## EdgeOne 部署步骤

### 1. 登录 EdgeOne 控制台

访问 [EdgeOne 控制台](https://console.cloud.tencent.com/edgeone) 并登录您的账号。

### 2. 创建站点

1. 点击「站点管理」→「添加站点」
2. 输入您的域名，点击「下一步」
3. 按照提示完成域名验证和 DNS 配置

### 3. 配置源站

1. 在「站点管理」中选择您的站点
2. 点击「源站管理」→「添加源站」
3. 选择「静态网站托管」
4. 上传构建生成的 `out` 目录下的所有文件

### 4. 配置路由规则

1. 点击「规则引擎」→「路由规则」
2. 添加以下路由规则：

| 路径模式 | 匹配类型 | 处理方式 |
|---------|---------|--------|
| `/*` | 前缀匹配 | 静态文件 |
| `/` | 精确匹配 | 静态文件 (index.html) |
| `404` | 状态码 | 静态文件 (404.html) |

### 5. 配置缓存规则

1. 点击「缓存配置」→「缓存规则」
2. 添加以下缓存规则：

| 路径模式 | 缓存时间 | 缓存策略 |
|---------|---------|--------|
| `/*.js` | 30天 | 遵循源站 |
| `/*.css` | 30天 | 遵循源站 |
| `/*.png` | 30天 | 遵循源站 |
| `/*.jpg` | 30天 | 遵循源站 |
| `/*.svg` | 30天 | 遵循源站 |

### 6. 配置安全规则

1. 点击「安全配置」→「WAF 规则」
2. 确保以下规则不会阻止网站正常运行：
   - 关闭对静态资源的过度防护
   - 允许必要的 JavaScript 执行
   - 允许跨域资源共享 (CORS) 

### 7. 配置 HTTPS

1. 点击「HTTPS 配置」→「证书管理」
2. 申请或上传 SSL 证书
3. 启用 HTTPS 并强制 HTTPS 访问

## 常见问题及解决方案

### 1. 页面加载空白

**原因**：路由配置不正确，导致静态文件无法找到。

**解决方案**：确保路由规则正确配置，特别是 `/*` 路径的处理方式。

### 2. 静态资源加载失败

**原因**：资源路径配置不正确，或者缓存规则导致资源无法更新。

**解决方案**：检查资源路径是否正确，清除 EdgeOne 缓存。

### 3. JavaScript 执行错误

**原因**：EdgeOne 的安全规则阻止了 JavaScript 执行。

**解决方案**：调整安全规则，允许必要的 JavaScript 执行。

### 4. 页面样式错乱

**原因**：CSS 文件加载失败或缓存问题。

**解决方案**：检查 CSS 文件路径，清除 EdgeOne 缓存。
